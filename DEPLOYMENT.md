# Deployment Guide for Railway

This guide will help you deploy the Team Task Manager application to Railway.

## Prerequisites

- GitHub account
- Railway account (sign up at https://railway.app)
- Git installed locally

## Step-by-Step Deployment

### 1. Prepare Your Repository

1. Initialize git repository (if not already done):
```bash
git init
git add .
git commit -m "Initial commit"
```

2. Create a GitHub repository and push your code:
```bash
git remote add origin <your-github-repo-url>
git branch -M main
git push -u origin main
```

### 2. Deploy to Railway

#### A. Create New Project

1. Go to [Railway.app](https://railway.app)
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Authorize Railway to access your GitHub
5. Select your repository

#### B. Add PostgreSQL Database

1. In your Railway project, click "New"
2. Select "Database" → "PostgreSQL"
3. Railway will automatically provision a PostgreSQL database
4. The `DATABASE_URL` will be automatically available as an environment variable

#### C. Configure Environment Variables

1. Click on your service (not the database)
2. Go to "Variables" tab
3. Add the following variables:

```
DATABASE_URL=${{Postgres.DATABASE_URL}}
JWT_SECRET=your-super-secret-jwt-key-change-this-to-something-random
NODE_ENV=production
PORT=5000
CLIENT_URL=https://your-app-name.up.railway.app
```

**Important:** 
- Replace `JWT_SECRET` with a strong random string
- `DATABASE_URL` will automatically reference your PostgreSQL database
- `CLIENT_URL` will be your Railway app URL (you can update this after first deployment)

#### D. Configure Build Settings

Railway should auto-detect your configuration from `nixpacks.toml`, but verify:

1. Go to "Settings" tab
2. Check "Build Command": Should be auto-detected
3. Check "Start Command": Should be `npm start`

#### E. Deploy

1. Railway will automatically deploy when you push to GitHub
2. Wait for the build to complete (usually 2-5 minutes)
3. Once deployed, Railway will provide you with a URL like: `https://your-app-name.up.railway.app`

### 3. Run Database Migrations

After first deployment, you need to run migrations:

1. In Railway dashboard, click on your service
2. Go to "Settings" tab
3. Scroll to "Deploy Triggers"
4. Or use Railway CLI:

```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Link to your project
railway link

# Run migrations
railway run npx prisma migrate deploy
```

### 4. Update CLIENT_URL

1. Copy your Railway app URL
2. Go back to "Variables" tab
3. Update `CLIENT_URL` with your actual Railway URL
4. The app will automatically redeploy

### 5. Test Your Deployment

1. Visit your Railway URL
2. Try signing up a new user
3. Create a project
4. Create tasks
5. Test all features

## Frontend Deployment (Optional - Separate Frontend)

If you want to deploy the frontend separately on Vercel or Netlify:

### Vercel Deployment

1. Go to [Vercel](https://vercel.com)
2. Click "New Project"
3. Import your GitHub repository
4. Configure:
   - **Root Directory**: `client`
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
5. Add environment variable:
   ```
   VITE_API_URL=https://your-railway-backend.up.railway.app
   ```
6. Deploy

### Update Backend CORS

Update `CLIENT_URL` in Railway to your Vercel URL:
```
CLIENT_URL=https://your-app.vercel.app
```

## Troubleshooting

### Build Fails

1. Check build logs in Railway dashboard
2. Ensure all dependencies are in `package.json`
3. Verify `nixpacks.toml` configuration

### Database Connection Issues

1. Verify `DATABASE_URL` is set correctly
2. Check if Prisma migrations ran successfully
3. Run migrations manually: `railway run npx prisma migrate deploy`

### Application Crashes

1. Check application logs in Railway dashboard
2. Verify all environment variables are set
3. Check for any missing dependencies

### CORS Errors

1. Verify `CLIENT_URL` matches your frontend URL
2. Check CORS configuration in `server/index.js`

## Updating Your Application

1. Make changes locally
2. Commit and push to GitHub:
```bash
git add .
git commit -m "Your update message"
git push
```
3. Railway will automatically detect changes and redeploy

## Environment Variables Reference

| Variable | Description | Example |
|----------|-------------|---------|
| DATABASE_URL | PostgreSQL connection string | Auto-generated by Railway |
| JWT_SECRET | Secret key for JWT tokens | Random string (min 32 chars) |
| NODE_ENV | Environment mode | production |
| PORT | Server port | 5000 |
| CLIENT_URL | Frontend URL for CORS | https://your-app.up.railway.app |

## Security Checklist

- ✅ Change JWT_SECRET to a strong random value
- ✅ Enable HTTPS (automatic on Railway)
- ✅ Set NODE_ENV to production
- ✅ Configure proper CORS origins
- ✅ Keep dependencies updated
- ✅ Don't commit .env file
